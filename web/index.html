OpenLogiTwin

Conveyor Sorting Cell — a virtual PLC commissioning rig

A narrow, demo-ready, technically credible digital twin of a single conveyor sorting cell: real industrial protocols, a deterministic plant model, and a soft-PLC you can swap for OpenPLC. Built for PLC training and virtual commissioning — not a generic factory sim.

🖥 Open the live HMI ▶ View the demo report ★ GitHub repository

The HMI and demo report are regenerated from the scenario suite on every push to main.

What's inside

🖥

Web HMI (ISA-101)

A zero-install, high-performance operator console: replays deterministic traces, or goes live over a stdlib WebSocket to drive the running twin.

🔌

Real protocols

Hand-rolled, standards-compliant Modbus TCP server/client (+ pymodbus), real OPC UA (asyncua), and MQTT telemetry.

🎯

Deterministic

Fixed-dt lock-step plant model. Scenarios are bit-reproducible — same input, same result, every run.

🧩

PLC-ready

Swappable soft-PLC with a sample IEC 61131-3 Structured Text program for OpenPLC. A tag registry is the single source of truth.

📦

Multi-parcel FIFO cell

An 8-slot destination ring sorts dense streams correctly where a single-register cell mis-routes them.

📊

Telemetry

Every cycle, sort, fault and state change logged to SQLite with CSV/JSON export and an optional live MQTT sink.

✅

Tested & cheap

28 dual-mode test files; CI matrix across Python 3.9–3.13 on Ubuntu & Windows. The core needs zero third-party packages.

Quickstart

git clone https://github.com/aydogandagidir/conveyor-sorting-twin
cd conveyor-sorting-twin

python scripts/run_tests.py              # the full suite (stdlib only)
python scripts/scenario_manager.py run-all   # run every scenario with expectations
python scripts/run_full_demo.py --export-dir=_site   # build this demo report
